## Decisions

### D-001 — One parent per player (MVP)
Maximum one parent/guardian account per player profile at MVP. Multi-guardian support (e.g., co-parents) is deferred. Rationale: keeps access control model simple for launch.

### D-002 — Germany-only geographic scope (MVP)
EUR currency, EU data residency, German market only at launch. Multi-region deferred.

---

### D-003 — Coach profile remains on Scout tier after subscription lapse
When a coach's paid subscription lapses, their public profile remains listed on the marketplace under Scout tier capabilities. They do not become hidden.

### D-004 — Google/Apple calendar sync is post-MVP
MVP availability management is handled via an in-app calendar view. Two-way external calendar sync deferred to post-MVP.

### D-005 — DISPUTED state resolution logic confirmed
Admin resolves in coach's favour → session moves to COMPLETED, credit deducted. Admin resolves in parent's favour → session moves to REFUNDED.

### D-006 — Coach score normalisation (FR-DEV-010) is post-MVP
Raw scores are used at launch. Cross-coach normalisation requires sufficient ecosystem data and is deferred.

### D-007 — Development Correlation Engine gated at Academy tier
FR-DEV-011 confirmed as Academy (Elite) only.

### D-016 — Video retention for non-yearly subscribers is 30 days
playerPortal.md (30 days) is the authoritative value. videoModule.md and functional-requirements.md references to 90 days are superseded.

### D-008 — Unified Timeline access: configurable expiry period
Coach access to a player's timeline expires after a configurable period of inactivity (no active paid session relationship). The expiry duration is stored in the database and must be changeable without a code deployment.

### D-009 — Feature gates must be configurable, not hardcoded
All tier-based feature gates (e.g., which features are available on Scout / Instructor / Academy / player tiers) must be configurable via the database or a config layer. No feature gate logic should be hardcoded. This enables tier adjustments without code changes.

---

### D-014 — App UI languages: English and German
The platform supports English and German as UI languages at MVP.

### D-015 — First Touch skill code: F1T (not FIT)
The correct code for First Touch in the 15-skill taxonomy is F1T. Applied everywhere the taxonomy appears.
